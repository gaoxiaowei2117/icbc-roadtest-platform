# 用户使用指南

## 1. 登录入口

打开 `https://gogoxoxo.duckdns.org:9443/booking/`。

新用户先注册，输入邮箱收到的 6 位验证码完成验证，再登录系统。验证码 10 分钟内有效；没有收到时可在验证页面重新发送，并检查垃圾邮件目录。

## 2. 完善抢号资料

进入“资料”页面填写：

- ICBC 驾照号和姓氏
- 考试类型
- 一个或多个考点
- 可接受的最早、最晚日期
- 时间范围、星期和上午/下午偏好

进入“设置”页面保存 ICBC keyword。keyword 通过 HTTPS 提交给后端，并立即使用 worker 公钥加密；云端数据库只保存密文，后端不持有解密私钥。解密私钥只在本地 worker 中。

资料或 keyword 不完整时不能创建任务。修改抢号条件前，建议先取消正在运行的旧任务，再保存资料并创建新任务。

## 3. 创建和停止任务

进入“任务”页面创建任务。每个用户同时只能存在一个 `pending` 或 `running` 任务。

任务状态含义：

| 状态 | 含义 |
|---|---|
| `pending` | 等待 worker 领取，或一个 600 秒执行周期结束后正在排队续跑 |
| `running` | worker 正在查询或处理预约 |
| `done` | 已预约成功，任务结束 |
| `failed` | 遇到不可重试错误，需要根据错误修复后新建任务 |
| `cancelled` | 用户已取消，worker 会在下一次状态检查时停止 |

点击取消后，已经发出的单次网络请求不能被瞬间撤回，但 worker 会在本轮检查结束后停止，不再继续下一轮。

## 4. 判断任务是否仍在工作

任务页面会自动刷新以下信息：

- **领取次数 (`attempt_count`)**：worker 领取或重新领取任务的次数。600 秒周期结束后自动重排会使它增加。
- **查询轮次 (`progress_rounds`)**：worker 向后端成功上报的考点查询次数。这个值持续增加，说明任务仍在运行。
- **最近进度**：例如 `考点 274 查询结果 no_appointments`，表示该考点本轮没有符合条件的预约。
- **更新时间**：最近一次进度上报时间。

领取次数和查询轮次不是同一个概念，因此数值通常不同。多个考点每查询一个考点都会增加一次页面查询计数。

默认情况下，没有合适考位时会随机等待 12-20 秒再开始下一轮。页面短时间没有变化不一定是异常；如果超过 15 分钟没有任何进度，后端会把可能失联的 `running` 任务重置为 `pending`，等待 worker 重新领取。

## 5. 常见错误

### Gmail 认证失败

日志出现 `AUTHENTICATIONFAILED Invalid credentials`，通常表示 worker 的 `GMAIL_EMAIL` 或 `GMAIL_APP_PASSWORD` 无效。必须使用开启两步验证后生成的 Gmail 应用专用密码，不能使用普通登录密码。

### 凭据解密失败

本地 worker 的 `SECRET_PRIVATE_KEY` 必须与云端 `SECRET_PUBLIC_KEY` 属于同一密钥对。更换密钥对后，用户需要在“设置”页面重新保存 keyword。

### 一直显示 pending

通常表示 worker 未启动、无法访问云端 API，或 `WORKER_API_KEY` 不一致。该问题需要管理员查看 worker 状态和日志。

### 显示 no_appointments

这不是程序错误，只表示本次查询没有符合日期、考点和时段偏好的考位。任务会继续查询。

## 6. 隐私与使用边界

- 不要在聊天、截图或日志中公开驾照号、keyword、验证码、Gmail 应用密码或任何 `.env` 内容。
- SealedBox 保护数据库中已经保存的 keyword；它不能替代 HTTPS，也不能防止已经控制云端应用进程的攻击者截获用户之后提交的新数据。
- `DRY_RUN=false` 时 worker 会真实锁号和提交预约；测试环境应使用 `DRY_RUN=true`。
- 同时运行多个 worker 会提高并发处理能力，但也会增加对 ICBC 和 Gmail 的请求量，不应通过大量实例绕过对方服务限制。
