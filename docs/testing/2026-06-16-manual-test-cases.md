# 手动测试用例 — 网页驱动抢号（多用户化）

- 日期：2026-06-16
- 范围：注册/登录 → 档案配置 → 建任务 → worker 抢号 的端到端流程，含本轮三个修复（保存弹窗、日期/时间校验、同账户限一个活跃任务）。

## 环境准备

| 项 | 状态/命令 |
|---|---|
| PostgreSQL | docker `icbc-pg` 运行中 |
| 后端 | `cd backend && .venv/bin/uvicorn app.main:app --port 8000`（迁移已到 0002）|
| 前端 | `cd frontend && npm run dev` → `http://127.0.0.1:5173/booking/` |
| worker（真实抢号才需要）| `worker/.env` 配 `SECRET_PRIVATE_KEY`（与后端公钥成对）+ `GMAIL_EMAIL`/`GMAIL_APP_PASSWORD`；`worker/config.yml` 系统级 base |

## 模块 A：账户

### TC-A1 注册并自动登录
- 步骤：打开 `/booking/` → 注册新邮箱 + 密码 → 提交
- 预期：注册成功，自动登录，进入面板；刷新后仍登录（refresh token 续期）

### TC-A2 重复邮箱注册
- 步骤：用已注册邮箱再次注册
- 预期：报错"邮箱已注册"（409）

### TC-A3 错误密码登录
- 步骤：登出 → 用错误密码登录
- 预期：报错"邮箱或密码错误"（401）

---

## 模块 B：档案配置 + 校验 + 弹窗（本轮修复）

### TC-B1 完整档案保存成功（弹窗）
- 前置：已登录
- 步骤：设置页填 驾照号、姓氏、考试类型、时间区间(09:00–17:00)、日期区间、考点(单选)、星期、上下午 → 点「保存资料」
- 预期：**弹窗"资料已保存"**；刷新页面后各字段回显一致

### TC-B2 结束日期早于开始日期 → 拦截（弹窗）
- 步骤：期望最早日期 = 2026-08-01，期望最晚日期 = 2026-07-01 → 保存
- 预期：**弹窗"结束日期不能早于开始日期"**，不提交（后端无更新）

### TC-B3 结束时间早于/等于开始时间 → 拦截（弹窗）
- 步骤：时间区间 开始 = 17:00，结束 = 09:00（或两者相等）→ 保存
- 预期：**弹窗"结束时间必须晚于开始时间"**，不提交

### TC-B4 考点单选 + 显示号
- 步骤：展开考点下拉
- 预期：**单选**下拉（不能多选）；每项显示「**名称（pos_id）**」，如 `Vancouver...（271）`

### TC-B5 保存失败弹窗
- 步骤：（构造失败，如断网或后端停）保存资料
- 预期：**弹窗"保存失败：…"**

---

## 模块 C：keyword（凭据）

### TC-C1 保存 keyword（弹窗 + 加密）
- 步骤：keyword 区填值 → 保存
- 预期：**弹窗"keyword 已加密保存"**；区域显示"✓ 已配置"
- 后端核验（可选）：DB `secret.ciphertext` 是密文，不含明文 keyword

### TC-C2 删除 keyword
- 步骤：点删除 → 确认
- 预期：弹窗"keyword 已删除"；"✓ 已配置"消失

---

## 模块 D：建任务（前置校验 + 限一个活跃任务，本轮修复）

### TC-D1 档案不全 → 建任务被拒
- 前置：新账户只注册、未填档案
- 步骤：抢约任务页点「创建任务」
- 预期：**弹窗"创建失败：请先在设置页补全：…"**（400），列出缺失项

### TC-D2 档案完整 → 建任务成功
- 前置：档案 + keyword 都配好
- 步骤：创建任务
- 预期：**弹窗"任务已创建…"**；任务列表出现一条 `pending`

### TC-D3 同账户同时只能一个活跃任务（核心）
- 前置：已有一个 `pending`/`running` 任务
- 步骤：再次点「创建任务」
- 预期：**弹窗"创建失败：已有进行中的任务，完成或取消后再新建"**（409）；列表仍只有一条

### TC-D4 取消后可再建
- 步骤：取消现有任务 → 状态变 `cancelled` → 再创建任务
- 预期：新任务创建成功（201）

### TC-D5 只看到自己的任务
- 步骤：用账户 X 建任务 → 登出 → 用账户 Y 登录看任务列表
- 预期：Y 看不到 X 的任务

## 模块 E：worker 抢号（⚠️ 真实操作，需准备）

> worker 已支持系统级 `DRY_RUN`。联调必须先使用 `DRY_RUN=true`，此时只登录和查号，不锁号、不下单、不修改 ICBC 邮箱；确认配置后再用 `DRY_RUN=false` 测试真实预约。

### TC-E1 worker 认领并用网页档案抢号
- 前置：worker `.env`/`config.yml` 配好；某账户档案 + keyword 真实有效；有一个 `pending` 任务
- 步骤：启动 worker
- 预期：worker 日志显示 claim 到任务 → 解密 keyword → 用该账户档案（考点/日期/时间/偏好）构造 config → 登录 ICBC → 查号；有合适号时，`DRY_RUN=false` 才会锁号、收 OTP、下单并回报 `done`。无号时持续查询，600 秒后重排继续，不应标记为 `failed`。

### TC-E2 任务卡死 reaper
- 前置：worker 中途崩溃，任务停在 `running`
- 预期：连续超过 `RUNNING_TIMEOUT_MINUTES`（默认 15）没有进度心跳后，后端 reaper 把任务重置为 `pending`；持续有 `last_progress_at` 更新的长任务不应被重置
